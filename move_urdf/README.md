# ROS2 Xacro Robot Modeling Practice (move_urdf)

이 패키지는 URDF의 한계를 극복하기 위해 매크로 언어인 Xacro(XML Macros)를 도입하여, 복잡한 로봇 모델을 효율적으로 설계하고 유지보수하는 방법을 실습한 기록입니다.

## 1. 수업 내용 요약

### Xacro의 정의 및 필요성
- **정의**: URDF를 효율적으로 작성하기 위한 매크로 언어로, 최종적으로는 사전 처리 과정을 거쳐 URDF 파일로 변환됩니다.
- **필요성**: 
  - 동일한 형태의 Link나 Joint가 반복될 때 코드를 획기적으로 줄일 수 있습니다.
  - 로봇의 크기나 반지름 등 수치 수정이 필요할 때 일일이 찾아서 고칠 필요 없이 변수 처리가 가능합니다.
  - 로봇 모델이 복잡해질수록 가독성과 관리 효율을 높여줍니다.

### 주요 기능
- **xacro:property (변수화)**: 공통으로 사용하는 수치(길이, 반지름 등)를 변수로 정의하여 한 곳에서 관리합니다.
- **xacro:macro (함수화)**: 반복되는 구조(예: 로봇의 여러 다리)를 블록 단위로 정의하여 재사용합니다.
- **수학적 연산**: 변수 간의 사칙연산을 지원하여 모델의 비례를 동적으로 계산할 수 있습니다.

## 2. 실습 내용

본 실습에서는 `move_urdf` 워크스페이스에서 `urdf_r2d2` 패키지를 활용하여 Xacro 기반의 R2D2 로봇 모델을 구동했습니다.

### 주요 구현 사항 (r2d2.urdf.xacro)
1. **패키지 구성**: `urdf_r2d2` 패키지 내부에 Xacro 파일을 배치하고, 런치 파일을 통해 실시간으로 URDF로 변환하도록 설정했습니다.
2. **모델 최적화**: 로봇의 머리(head) 높이나 다리(leg) 간격을 Xacro 변수(`property`)로 선언하여 수치 수정을 용이하게 했습니다.
3. **매크로 활용**: 좌우 대칭인 다리 구조를 `macro`로 정의하여 코드 중복을 최소화했습니다.
4. **시각화 연동**: `demo_launch.py`를 통해 `robot_state_publisher`가 Xacro를 해석하여 RViz2에 로봇 모델을 성공적으로 띄우도록 구성했습니다.

### 실행 절차 및 검증

작성한 Xacro 모델을 빌드하고 런치 파일을 통해 동작을 확인했습니다.

```bash
# 1. 패키지 빌드 및 환경 활성화
cd ~/move_urdf
colcon build
source install/local_setup.bash

# 2. Xacro 포함 런치 파일 실행
ros2 launch urdf_r2d2 demo_launch.py

# 3. RViz2 검증
# - RViz 창에서 'RobotModel'이 정상적으로 표시되는지 확인
# - Fixed Frame이 'odom' 또는 모델에 맞는 기준으로 설정되어 있는지 확인
# - Xacro에서 정의한 변수값(높이, 간격 등)이 모델에 올바르게 반영되었는지 시각적으로 검토
