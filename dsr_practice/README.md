# ROS2 Doosan Robotics Control Practice (dsr_practice)

이 패키지는 두산 로보틱스(Doosan Robotics) 인터페이스를 활용하여 가상 및 실물 로봇을 제어하고, MoveIt2 Python API와 STT(음성 인식) 기술을 결합하여 지능형 로봇 제어 시스템을 구축한 실습 기록입니다.

---

## [12차시] Doosan 로봇 개발 환경 구축 (25장)

### 1. 주요 패키지 및 환경
- **주요 패키지**: `dsr_bringup2`, `dsr_description2`
- **개발 환경**: Docker 컨테이너(Ubuntu 22.04 + ROS2 Humble) 기반의 샌드박스 환경

### 2. 수업 및 실습 요약
- **Docker 기반 샌드박스 구성**: 로봇 제조사마다 요구하는 OS 버전과 종속성 라이브러리가 다르기 때문에, 호스트 OS의 환경을 보존하면서 독립적인 개발 환경을 보장하기 위해 Docker 컨테이너를 사용함.
- **실행 모드 전환**: `virtual` 모드(가상 시뮬레이션)에서 알고리즘을 선검증한 후, 동일한 코드를 `real` 모드(실물 로봇)에 적용하여 하드웨어 추상화 계층(Hardware Abstraction Layer)의 효율성을 이해함.
- **통신 계층 설정**: 로봇 컨트롤러와 PC 간의 TCP/IP 통신을 설정하고, ROS2 환경에서 실제 로봇의 상태 데이터(Joint State 등)가 정상적으로 수신되는지 확인 작업을 수행함.

## [13차시] MoveIt2 Python 제어 및 Pick & Place (26, 27, 28장)

### 1. 주요 파일 및 구현 내용
- **`mp_basic.py` (기초 포즈 제어)**
  - **구현 내용**: `MoveItPy` 라이브러리를 선언하고 `get_planning_component`를 통해 로봇의 목표 자세를 설정함. `plan()`과 `execute()` 명령을 사용하여 Joint 및 Pose 목표값으로 로봇을 구동함.
- **`mp_waypoint.py` (플래너 비교 및 경로 생성)**
  - **구현 내용**: **OMPL**(샘플링 기반)과 **Pilz Industrial Motion Planner**(결정론적 기반)의 특징을 비교 실습함. 단순 이동은 연산이 빠른 OMPL을, 정밀한 직선 운동이 필요한 구간에는 `LIN` 플래너를 적용하도록 코드를 분기 작성함.
- **`pick_and_place.py` (통합 공정 구현)**
  - **구현 내용**: TF 좌표계를 활용해 물체의 위치를 실시간 파악하고, `pymodbus` 라이브러리로 그리퍼를 제어하여 물체를 집고 옮기는 전체 공정을 자동화함.

### 2. 수업 및 실습 요약
- **플래너(Planner)의 필요성**: 로봇이 목표지점까지 이동할 때 단순히 관절값을 보간하여 움직이면 자신의 몸체나 주변 장애물에 부딪힐 수 있음. 따라서 장애물을 감지하고 충돌이 없는 최적의 경로를 실시간으로 계산해주는 '두뇌' 역할의 플래너가 필수적임.
- **Move Group Interface**: 복잡한 역기구학(Inverse Kinematics) 수식을 개발자가 직접 계산할 필요 없이, 엔드이펙터의 좌표(x, y, z)만 입력하면 로봇이 스스로 관절 각도를 도출하여 움직이는 고수준 제어 방식을 습득함.
- **이기종 장비 통합 (Modbus TCP)**: 산업용 그리퍼에서 표준으로 사용되는 Modbus TCP 프로토콜을 파이썬 코드로 직접 구현하여, ROS2 기반의 로봇 팔과 외부 장치(그리퍼) 간의 유기적인 연동 제어를 수행함.

## [17차시] STT 연동 음성 제어 시스템 (31장)

### 1. 주요 파일 및 구현 내용
- **`stt_node.py` (음성 인식 및 발행)**
  - **구현 내용**: 마이크 입력을 실시간으로 캡처하여 Google Web Speech API로 전송하고, 반환된 텍스트에서 로봇 동작 트리거를 위한 핵심 키워드("픽", "플레이스")를 필터링함.
  - **구현 방법**: 감지된 단어를 `std_msgs/String` 메시지에 담아 `/stt_result` 토픽으로 즉시 발행하도록 구성함.
- **`stt_pick_and_place.launch.py` (시스템 통합)**
  - **구현 내용**: STT 노드와 로봇 제어 노드를 하나의 런치 파일로 통합 실행함. `stt_node`가 발행하는 토픽을 제어 노드가 구독(Subscribe)하게 하여 인간의 음성이 로봇 동작의 트리거가 되는 이벤트 기반 시스템을 완성함.

### 2. 수업 및 실습 요약
- **인간-로봇 상호작용(HRI)**: 마우스나 키보드 같은 전통적인 입력 수단에서 벗어나 인간의 자연어를 로봇 제어 신호로 변환하는 사용자 친화적 인터페이스 구축 프로세스를 학습함.
- **비동기 이벤트 처리**: 프로그램이 무한 루프를 돌며 대기하는 방식이 아니라, 음성이 인식되는 시점에만 콜백 함수가 실행되어 로봇의 동작을 트리거하는 효율적인 비동기 제어 로직을 실습함.
- **실무적 노이즈 처리**: 주변 소음 임계값(`energy_threshold`) 파라미터를 조정하여 다양한 작업 환경에서도 음성 명령의 인식 정확도를 높이는 최적화 기법을 습득함.

## 실행 절차 및 검증

```bash
# 1. 두산 로봇 실행 (가상 모드 기준)
ros2 launch dsr_bringup2 dsr_bringup2_moveit.launch.py mode:=virtual model:=m0609 host:=127.0.0.1

# 2. Pick & Place 통합 공정 실행
ros2 run dsr_practice pick_and_place

# 3. STT 기반 음성 제어 시스템 실행
ros2 launch dsr_practice stt_pick_and_place.launch.py

# 검증 포인트
# - mp_basic: 로봇이 충돌 없이 지정된 홈 포즈로 정확하게 복귀하는지 확인
# - pick_and_place: 그리퍼가 Modbus 명령을 정상 수신하여 물체를 파지하고 놓는지 확인
# - STT: "픽" 명령 시 인식 로그가 출력되고 로봇이 즉각 동작을 시작하는지 확인
